# Privacy Gate ‚Äî Status & Trace

On-device de-identification for the prompt‚Üíchart app. The user's prompt and dataset
are stripped of identifying content **before** anything is sent to the cloud LLM; the
chart is computed locally. This file is the single source of truth (it replaces the
old `PRIVACY_GATE_PLAN*.md`).

Status: **built, tested, black-box red-teamed, and white-box audited.** All real findings
fixed. Nothing committed to git yet at the time of writing (see "How to commit" below).

---

## 1. The guarantee (plain English)

Every word that leaves the device in the user payload is one of:
- an **opaque token** (`col_N` = a column name, `val_N` = a category value, `lit_N` = a redacted literal), or
- an **allowlisted word** ‚Äî common English/Turkish vocabulary + chart vocabulary (~7,900 words), or
- known **frame scaffolding** (our own template + chart-spec keywords).

Anything else ‚Äî names, PII, account/ID numbers, rare/specific strings, misspellings ‚Äî is
**masked to a placeholder**. A runtime **tripwire** (`verifyOutbound`) re-checks the whole
outbound message right before the network call and **fails closed** on any violation.
The map back from tokens to real values (`toReal`) never leaves the device.

This holds **automatically**, with no reliance on the user reviewing anything (important:
the dev-facing "Sent to AI" card is being removed ‚Äî the protection does not depend on it).

---

## 2. Threat model

**In scope:** an *honest user's mistake* ‚Äî accidentally typing/pasting a name, phone, ID,
account number, email, or a data dump into the prompt.

**Out of scope (explicit owner decisions):**
- **Sophisticated adversarial evasion.** We defeat the common/honest cases (Unicode
  separators, fullwidth, homoglyphs, token-lookalikes) but do not claim resistance to a
  determined attacker crafting input to leak their *own* data.
- **API key / egress.** The public demo intentionally runs on the browser-held Anthropic
  key; dev/testing uses the faithful OAuth proxy (`scripts/cli-proxy.mjs`). Key rotation /
  history purge are out of scope. (Use a scoped, low-limit, disposable demo key.)
- **Structural inference** (cardinality/row-count leakage). Profiling stats are *banded*
  (`low/medium/high-cardinality`, `some missing`), not raw, which blunts this; full
  resistance is declined.

## 3. How it works ‚Äî the de-identification pipeline

`deidentify(prompt, rows)` in `src/ai.ts`, in order:

1. `normalizeLite` ‚Äî NFKC; strip zero-width chars; fold Unicode spaces‚Üíspace and
   dashes/minus/middot‚Üí`-`; **cap input at 4000 chars** (ReDoS guard; the dropped tail is
   never sent). Keeps Turkish letters intact.
2. `neutralizeTokenLookalikes` ‚Äî mask user-typed `col_/val_/lit_N`-shaped strings
   (case-insensitive) so they can't be confused with real tokens.
3. `maskPII` ‚Äî checksum-gated where possible: ISO dates, IBAN (mod-97), payment card
   (Luhn), Turkish national ID (TCKN), phone (‚â•10-digit run), **grouped digit identifiers**
   (‚â•2 groups joined by space/dash/dot/comma/colon/slash, ‚â•5 digits ‚Äî SSN/PIN/MAC/etc.), IPv4.
4. `tokenizeText` ‚Äî replace dataset **column names ‚Üí `col_N`** and **category values ‚Üí `val_N`**
   (only `col_/val_` terms; never re-matches `lit_`).
5. `fuzzyTokenizeResiduals` ‚Äî a residual word that's a near-miss (bounded edit distance) of a
   column/value maps to **that token** (rescues typos like `temprature`‚Üítemperature). Output
   is always a token or a mask ‚Äî never verbatim.
6. `redactLiterals` ‚Äî emails (bounded), URLs, ISO dates, grouped/separated/decimal numbers,
   and ‚â•4-digit numbers ‚Üí `lit_N`.
7. `scrub` = `enforceAllowlist` ‚Äî **default-deny**: any residual word (incl. digit-led
   alphanumerics like `4521A`) not in the allowlist ‚Üí `lit_N`. Bare numbers (`3`, `605`) pass
   per the number policy (top-N / thresholds; ‚â•4-digit already masked above).
8. `buildSpecMessage` + `enforceOutbound` ‚Äî assemble the message (prompt + banded schema +
   template) and mask anything outside tokens/allowlist/FRAME.
9. At egress, `claudeComplete` runs `verifyOutbound` and **throws (fails closed)** if anything
   un-allowlisted remains. The two repair messages are enforced + verified the same way.

Number policy: 1‚Äì3 digit bare numbers are kept (chart quantities); ‚â•4-digit and grouped
identifiers are masked. The tripwire treats bare numbers as allowed by design.

## 4. What was built (increments)

1. **PII silent-mask + scale.** Phone/IBAN/card/TCKN/IP masking; value index built from a
   bounded **5,000-row sample** (not all rows) and **memoized once per dataset** ‚Äî per-keystroke
   work is O(prompt), independent of dataset size.
2. **Closed vocab allowlist + default-deny + tripwire.** `src/vocab.ts` (~7,900 EN+TR words,
   generated by workflow + iterated against 498 prompts); `enforceAllowlist` masks every
   non-allowlisted residual; `verifyAllowlist` proves it.
3. **Fuzzy-to-column tokenizer + allowlist audit.** Typos of columns/values map to their token
   (privacy-safe by construction). Audited the allowlist ‚Äî contains no person/place/brand names.
4. **Full-payload egress gate.** `FRAME` (template + chart-spec keywords), `verifyOutbound` /
   `enforceOutbound`; the *entire* message (incl. the schema half) is verified, fail-closed.
5. **Transparency UX (card-independent).** A standalone privacy panel: redaction summary
   chips, a **paste/blob block**, and a **name-risk warning**. Survives removal of the dev card.
6. **Web-Worker offload.** The one O(rows) step (chart aggregation) runs in a worker above a
   100k-row threshold; sync inline below it (zero overhead for normal data; sync fallback).
7. **Adversarial red-team** (8 classes, 192 cases) ‚Äî see bugs below.
8. **ReDoS/crash hardening + white-box code audit** (5 lenses) ‚Äî see bugs below.

## 5. Bugs found & fixed

Found during the build:
- **Phone/date fragmentation** ‚Äî `LONG_NUMBER` ate part of a multi-part phone/date range
  before PII detection could see the whole. Fixed by running PII/date detection first.
- **`normalizeLite` destroyed Turkish diacritics** (ƒ∞‚ÜíI, √ß‚Üíc) via NFD+strip. Fixed (NFKC only;
  combining-mark splitting is out of scope).

Found by the **red-team** (black-box):
- **Token-lookalike injection** ‚Äî `COL_8675309`, `Lit_5551234`, `cOl_99887766` leaked their
  digits (neutralizer was case-*sensitive*; tripwire shared the blind spot). Fixed: case-
  insensitive neutralization at the entry.
- **Separator-evasion of digit PII** ‚Äî `531‚àí24‚àí6789` (Unicode minus), `12¬∑345¬∑678` (middot),
  `531 24 6789` (NBSP), plus plain space/comma groups, split into sub-4-digit chunks that
  dodged every number rule. Fixed: Unicode-separator normalization + a grouped-digit rule.

Found by the **white-box audit + ReDoS stress test**:
- **Crash-DoS in `tokenizeText`** ‚Äî after PII masking collapsed a long number into one `lit_`,
  that value became a giant regex alternative ‚Üí "Invalid regular expression" ‚Üí `deidentify`
  threw. Fixed: `tokenizeText` excludes `lit_` entries.
- **Quadratic ReDoS in the email regex** (~1.1s/keystroke on a 50k single-line paste). Fixed:
  bounded quantifiers + the 4000-char input cap. Pathological inputs now <5ms.
- **Digit-led alphanumeric leak + tripwire blind spot** ‚Äî `4521A`, `14A`, `3X`, `0A:1B` left
  verbatim while `verifyOutbound` reported clean (the matcher was letter-initial only). Fixed:
  `WORD_RE` matches alphanumeric runs; bare numbers short-circuit so top-N/thresholds still pass.
- **Colon/slash numeric IDs** (MAC/IPv6/`12:34:56`) ‚Äî added `:`/`/` to the grouped-digit rule.

## 6. Known limitations / accepted residual risk

- **Dictionary-collision names/values.** A real name or high-cardinality value built from
  common words (`Rose Banks`, `Mark Day`, `Long Bridge Road`, a value `River`) is
  indistinguishable from ordinary language and passes. Mitigation: a **non-destructive
  name-risk warning** in the panel (2+ consecutive Title-Case words, excluding calendar ‚Äî 0
  false positives on the 498 prompts). We warn rather than mask (you can't tell a person
  `Rose Banks` from a place, and masking would break Title-Cased prose). The real fix is an
  ASK/HOLD-on-send UX (deferred).
- **Misspellings of non-schema words** (`thoses`, `teh`) mask ‚Äî a dictionary can't tell a typo
  from a name. Harmless filler; ~3/498 prompts, no accuracy impact observed.
- **Block is UI-level** ‚Äî `looksLikePastedData` disables Generate in the UI; a programmatic
  caller would bypass it, but the gate still de-identifies a blob's sensitive cells, so privacy
  holds (the block is UX, not the guarantee). A core-flow guard is a possible defense-in-depth add.

## 7. Test status & how to run

Counts at last run: **176 node unit + 88 vitest** green; tsc clean; production build OK.

Deterministic (no network):
```
node --experimental-strip-types --test "tests/*.test.ts"   # 176 pure unit tests
npx vitest run                                              # DOM + integration (88)
npx tsc --noEmit                                            # typecheck
```

Key test files (in `vitest/`):
- `gate.test.ts` ‚Äî the **tripwire**: full outbound message is token/vocab/frame-only across all 498 prompts + an adversarial mixed-leak case.
- `gate-unit.test.ts` ‚Äî pure gate functions (PII, fuzzy, redaction summary, paste/name detection, normalize).
- `redteam.test.ts` + `redteam-corpus.json` ‚Äî **permanent security regression suite** (192 adversarial cases; asserts tripwire=0 and no email/‚â•5-digit identifier survives).
- `redos.test.ts` ‚Äî performance/crash guard (pathological inputs <150ms, never throw).
- `worker.test.ts` ‚Äî the chart-aggregation offload (sync path + routing).
- `vocab-gaps.test.ts` ‚Äî diagnostic, gated behind `REPORT_VOCAB_GAPS=1` (shows what default-deny masks across the 498).

Live (real model, via the faithful OAuth proxy ‚Äî uses the Claude subscription, no API key):
```
PORT=8787 node scripts/cli-proxy.mjs &                                   # start proxy
RUN_LIVE_API=1 VITE_LLM_PROXY=http://localhost:8787 npx vitest run vitest/gate-live.test.ts
# full 498-case accuracy matrix:
RUN_LIVE_API=1 VITE_LLM_PROXY=http://localhost:8787 LIVE_CASES=vitest/live-cases.json npx vitest run vitest/live-matrix.test.ts
```
Live results: gate is byte-identical to baseline on ~485/498 prompts; gate-changed (typo)
prompts 15/15 correct; the de-id changes introduced **zero accuracy regression**.

## 8. Deferred / next steps

- **Rows-resident-in-worker** ‚Äî for true millions-of-rows scale, load/parse the dataset once
  inside the worker so rows aren't structure-cloned per submit (current worker is the groundwork).
- **ASK/HOLD-on-send UX** ‚Äî escalate the name-risk warning to an interactive confirm before send.
- **Core-flow paste guard** ‚Äî enforce `looksLikePastedData` in the request flow, not only the UI.
- **Fuzzy-to-calendar** ‚Äî rescue typo'd month/weekday refs the way column/value typos are rescued.
